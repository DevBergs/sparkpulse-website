Privacy Policy

Last updated: May 2025 · Effective immediately

Short version: We collect only what's needed to run the app. We never sell your data. You can delete everything at any time. This app is not a medical tool.

1. Who we are

Spark Pulse is an iOS application developed by Aivis Karlsbergs (Latvia, EU). For questions, contact: karlsbergsaivis@gmail.com

This app is not a medical device, diagnosis tool, therapy, or treatment. It is a focus and execution support tool.

2. What data we collect

We collect the following data when you use Spark Pulse:

Account data (if you sign in with Apple):

  • Email address (optional, provided by Apple)
  • Display name (optional, provided by Apple)
  • Firebase Auth UID (anonymous or linked)

App content data (only if cloud sync is enabled):

  • Daily logs — mood, energy, sleep hours, stress level, hydration, food types, notes
  • Tasks and reminders — titles, dates, completion status
  • Routines and activations — titles, categories, linked states, completion history
  • Focus sessions — duration, start time, session type
  • Ideas — text and voice memo transcriptions
  • Profile — display name, bio, avatar (emoji or photo)

Technical data (automatically collected):

  • Crash reports via Firebase Crashlytics
  • Anonymous app usage events (feature usage, session length) via Firebase Analytics
  • Device timezone (for hydration pacing)
  • App version

What we do NOT collect:

  • Location data
  • Health data (we do not use Apple HealthKit)
  • Contacts
  • Camera or microphone beyond in-app voice capture (stored locally only)
  • Advertising identifiers
  • Browsing history

3. Why we collect data

  • App functionality — tasks, routines, and daily logs are stored to provide the core app experience
  • Cloud sync — if enabled, your data syncs across devices via Firebase Firestore
  • Personalization — pattern detection for adaptive suggestions (processed on-device using your own data)
  • Crash fixing — anonymized crash reports help us identify and fix bugs
  • App improvement — anonymous feature usage analytics help us understand what works

4. Data storage and security

Data is stored in two places:

  • On device — using Apple SwiftData (encrypted by iOS device encryption)
  • Cloud — using Google Firebase Firestore (only if you enable sync). Data is isolated per user — no user can access another user's data. Firestore security rules enforce this at the database level.

All data transmission uses HTTPS/TLS. Firebase Auth sessions are secured by Google's infrastructure.

5. Third-party services

  • Firebase (Google) — Authentication, Firestore database, Crashlytics, Analytics. Firebase Privacy Policy
  • Apple — Sign in with Apple, App Store, device encryption. Apple Privacy Policy

We do not use any advertising networks, data brokers, or tracking SDKs.

6. Your rights (GDPR — EU users)

As a user in the European Union, you have the following rights:

  • Access — request a copy of your data
  • Correction — correct inaccurate data
  • Deletion — delete your account and all associated data
  • Portability — receive your data in a machine-readable format
  • Objection — object to processing for specific purposes
  • Withdrawal of consent — you can disable cloud sync at any time in Settings

To exercise any of these rights, email karlsbergsaivis@gmail.com or use the in-app deletion flow (Settings → Account → Delete Account).

7. Data deletion

You can delete your account and all data at any time:

  • In-app: Settings → Account → Delete Account & All Data
  • By email: Send deletion request to karlsbergsaivis@gmail.com
  • Web form: sparkpulse.app/delete-account

Deletion removes: Firebase Auth account, all Firestore documents, and local SwiftData on device. Crashlytics reports are anonymized and cannot be linked to you after deletion.

8. Data retention

Data is retained as long as you have an account. If you do not sign in for 12 consecutive months, we may delete inactive accounts with advance email notice (if email is available).

9. Children

Spark Pulse is not directed at children under 13. We do not knowingly collect data from children under 13. If you believe a child has provided us with personal data, contact us immediately.

10. Changes to this policy

If we make material changes to this policy, we will notify you via the app and update the "Last updated" date above. Continued use of the app after changes constitutes acceptance.

11. Contact

Questions or requests: karlsbergsaivis@gmail.com

Response time: within 30 days (GDPR requirement).
